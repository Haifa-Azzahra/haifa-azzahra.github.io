Improving IELTS Writing with Focus

A full-cycle Self-Regulated Learning experience — designed from Zimmerman's SRL framework outward, using a two-tool ecosystem where Storyline manages the session architecture and an AI writing coach provides adaptive, essay-specific feedback.

The Problem

IELTS Academic Writing is the section most likely to hold Band 6.5 learners back. Unlike listening or reading, there is no single correct answer — performance is judged across four interdependent criteria (Task Achievement, Coherence & Cohesion, Lexical Resource, and Grammatical Range & Accuracy), under 60-minute time pressure, with Task 2 carrying double the weight (British Council, ETS, IDP: IELTS Australia, 2020).

In Indonesia, many learners prepare independently due to cost barriers. Resources are widely available, but without structure, independent preparation tends to produce inconsistent practice, poor effort allocation, and a cycle of repeated test attempts — financial strain, declining confidence, reduced motivation.

🎯 Design question: How can you provide a self-directed learner with the structure of a coached session—without relying on a coach, an LMS, or a large budget?

Learning Needs Analysis

The core gap addressed here is not a lack of knowledge of IELTS criteria. Many Band 6.5 learners can identify assessment criteria, yet struggle to apply them consistently under test conditions.

This points to a gap in self-regulation: the ability to plan deliberately, monitor performance in real time, and reflect accurately to adapt.

Grounded in Zimmerman's Self-Regulated Learning (SRL) framework, this solution operationalises three phases — Forethought, Performance, and Reflection — not as content to be taught, but as a cycle to be practiced. Rather than explaining what SRL is, every scene in the module requires learners to do what self-regulated learners do.

Through repeated use, the cycle strengthens goal-setting, reflection accuracy, and metacognitive calibration — key conditions for score improvement. This shifts learners from passive practice to intentional, data-informed learning.

The Three-Phase Design

🔮 Phase 1

Forethought

Band gap analysis, scaffolded goal-setting with sentence stems, motivational grounding, and anxiety reframing. Learners diagnose before prescribing — identifying their own gap before setting a target.

🛠 Phase 2

Performance

Learners submit a full essay to the AI writing coach, receive targeted gap analysis, and manually name their primary gap — a deliberate design choice that makes metacognition the action, not a side effect.

🔄 Phase 3

Reflection

Three sequential cognitive moves: evaluate accurately, decide adaptively, commit specifically. Every prompt targets a specific self-regulatory behaviour with a theoretical basis. No generic "how did it go?" questions.

The Learning Ecosystem

⚙️ Articulate Storyline 3

Chosen for its persistent variable architecture — real-time learner-facing displays, conditional branching on learner-generated input, and session-to-session continuity via looping. The SRL scaffolding, session structure, and all 27 tracked variables live here.

🤖 IELTS AI Writing Coach

Analyses essays and delivers criterion-specific feedback, strategies, and practice. Feedback is intended as a reflective prompt — not a definitive score or examiner judgement.

Visit the Writing Coach
📊 JavaScript + Google Apps Script

Chosen as the evaluation layer because the tool was designed for LMS-free, zero-cost deployment. A Google Apps Script Web App receives all 27 session variables via XMLHttpRequest from Storyline, writing each completed session as a new row — enabling longitudinal SRL tracking without infrastructure cost.

📋 Google Sheets

The evaluation backend. Each completed session writes one row — timestamped, learner-named, with all Forethought, Performance, and Reflection variables stored in structured columns. Designed to surface SRL pattern changes across sessions.

Reflection

On the design process

The pivotal moment was refining the approach from teaching about SRL to designing through it. During the course outline matrix development, every scene was reframed: does this prompt action or reflection, or just comprehension? That single question restructured the entire learning architecture.

On the tool

The current version provides one practice prompt per session. A production version would expand prompt variety and invest further in the Storyline UI/UX — developing a visual language that reinforces self-regulation: calm, focused, and intentionally unhurried. The interface should signal that this is a space for reflection, not speed. The goal would also be better visual cohesion between the Storyline module and the Claude artifact so they feel like one ecosystem rather than two adjacent tools.

On what it means for learners

This learning experience builds the internal process of self-regulation. But self-regulation doesn't mean learning alone — writing improves through feedback, diverse models, and engagement with a learning community. This tool is one part of a larger ecosystem. It doesn't replace the external environment, and it was never designed to.
